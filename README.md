# CSP-Repair-Engine: Automated Formal Model Repair

This project is a Python-based automation pipeline designed to repair **Communicating Sequential Processes (CSP#)** models. It identifies inconsistencies in formal models based on failed assertions and uses **Gemini 2.5 Flash** to surgically fix the underlying logic.

By combining static analysis of CSP code with LLM-powered reasoning, this tool can automatically wrap state updates in `atomic` blocks, fix synchronization issues, and ensure that global variables like `coordinatorArray` remain consistent across processes.

---

## 🛠 Project Structure

The project is divided into four main modules:

* **`main.py`**: The orchestrator that manages the pipeline, reads error logs, and coordinates between the analyzer and the repair engine.
* **`analyser.py`**: Performs "Deep Scans" on CSP code, expands macros, and identifies which events affect variables involved in a failed assertion.
* **`engine.py`**: The interface with the Gemini API, handling prompt engineering, quota safety delays, and syntax post-processing.
* **`rules.py`**: A configuration file containing formal logic constraints (like atomicity and array consistency) to guide the LLM's repair strategy.

## 🚀 Getting Started

### 1. Prerequisites
* Python 3.8+
* A Google Gemini API Key
* **PAT (Process Analysis Toolkit)**: This project is specifically designed to repair models used within the PAT verification framework.

### 2. Installation
Clone the repository and install the required dependencies:

pip install google-generativeai python-dotenv

### 3. Environment Setup
Create a .env file in the root directory to store your API key (this file is ignored by Git for security):

GEMINI_API_KEY=your_actual_key_here

### 4. Running the Repair Pipeline
1. Place your .csp files in the models/ directory.
2. Ensure your mismatch_traces.json (generated by your model checker) is in the root directory.
3. Run the orchestrator:

python main.py

The repaired models will be saved in the repaired_models/ folder.

## 🔍 How It Works

* **Slicing**: The ProjectAnalyzer reads a failed assertion from the JSON log and identifies every process transition that modifies the problematic variables.
* **Prompting**: The repair_snippet function sends the broken code, the specific formal rule from rules.py, and the failed assertion to Gemini.
* **Syntax Guard**: The engine performs post-processing to remove markdown, fix double arrows (->), and ensure proper semicolon placement for CSP# syntax.
* **Surgical Replacement**: The updated code block is swapped back into the original file content without disturbing the rest of the model.
